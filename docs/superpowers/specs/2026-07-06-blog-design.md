# Blog Section + Admin Management — Design

**Date:** 2026-07-06
**Status:** Approved (design)

## Goal

A `/blog` section on the public site (list + one static page per post with its
own SEO) and a "Blog" view in the admin console to create/edit/delete posts.
Posts live in a Firestore `posts` collection; the public site reads them from
a published JSON mirror generated by the existing publish pipeline (never from
Firestore — bundle hygiene preserved).

## Decisions (agreed with user)

- **ES + EN per post**: each post carries both locales
  (`es`/`en: { title, excerpt, content }`); the public list/detail shows the
  active locale.
- **Markdown content**, rendered by a lightweight in-repo renderer (no new
  markdown dependency): `#`/`##`/`###` headings, paragraphs, `**bold**`,
  `*italic*`, `` `inline code` ``, fenced ``` code blocks, `- `/`1. ` lists,
  `> ` quotes, `[text](url)` links.
- **Blog is the 6th dock tab** (pencil-line SVG icon, label from CMS
  `nav.dock.blog`).
- Publishing model: saving a post writes Firestore; the existing **Publicar**
  button (deploy workflow) makes it live — same mental model as the rest of
  the CMS. The workflow's `content:pull` step picks the posts up at build time.
- One **starter post** (ES+EN welcome/launch post, status published) is seeded
  so the whole chain is verifiable end-to-end; Luis can edit or delete it.

## Architecture

### Data model (Firestore `posts/{slug}`)

```ts
export type PostLocaleContent = { title: string; excerpt: string; content: string };
export type BlogPost = {
  slug: string;              // doc id, kebab-case
  status: 'draft' | 'published';
  publishedAt: string;       // 'YYYY-MM-DD', editable
  tags: string[];
  es: PostLocaleContent;
  en: PostLocaleContent;
  createdAt?: unknown;       // serverTimestamp
  updatedAt?: unknown;
};
```

Types live in `src/content/posts-types.ts` (shared by admin, pull script and
public pages; kept out of `types.ts` because posts are not part of
`PortfolioContent`).

### Firestore rules

Add before the deny-all block (same owner guard as content):

```
    // Posts are edited only by the owner; the public site reads them from the
    // published JSON mirror, never from Firestore.
    match /posts/{doc} {
      allow read, write: if request.auth != null
        && request.auth.token.email_verified == true
        && request.auth.token.email == 'luis.atorred24@gmail.com';
    }
```

### Publish pipeline (`scripts/pull-content.ts` extension)

After pulling `content/{es,en}`:

1. Query `posts` where `status == 'published'`, order `publishedAt desc`
   (client-side sort to avoid composite index), strip timestamps, validate
   minimal shape (slug/status/publishedAt strings, es/en titles non-empty) →
   write `src/content/published/posts.json` (array; `[]` when none).
2. Regenerate `public/sitemap.xml`: the 5 existing routes + `/blog`
   (priority 0.8) + one `/blog/<slug>` per published post (priority 0.6,
   `lastmod` = publishedAt). The committed sitemap/baseline stays in sync
   because local pulls rewrite it too.

`src/content/published/posts.json` ships committed with the starter post
(mirroring Firestore, seeded by `scripts/migrate-blog-seed.ts`).

  ingeniería` / EN `Engineering notes & ideas` via `useI18n().locale`) —
  no CMS section for chrome (posts themselves are the content; avoids another

### Public — detail (`src/app/blog/[slug].tsx`)

- `export function generateStaticParams()` returns
  `posts.map(p => ({ slug: p.slug }))` from posts.json → one static HTML per
  post at export.
- Component: `useLocalSearchParams<{ slug }>`; find post in posts.json (not
  found → `router.replace('/blog')` effect + null render).
- Head (per post, active-locale ES for the static render): title
  `{es.title} — Luis De La Torre`, description `{es.excerpt}`, canonical
  `/blog/{slug}`, `og:type article`, og:title/description,
  `article:published_time`. `usePageTitle(title)` for SPA nav.
- Body (`PageShell`, `Container maxWidth` prose ~760): `← Blog` HoverLink →
  `/blog`; date mono accent + tags chips; H1 display 34; `<Markdown
  source={post[locale].content} />`; bottom `← Volver al blog`.

### Markdown renderer (`src/ui/markdown.tsx`)

`Markdown({ source })` — two-phase, no deps:

- **Blocks**: split lines; fenced ``` code blocks (surfaceCell bg, border,
  radius md, mono 13, padding 14, horizontal scroll via ScrollView
  horizontal); `#`/`##`/`###` → display 28/22/18 with top margins; `- ` and
  `1. ` list items (accent bullet/number + flex text); `> ` quote (accent left
  border 3, textMuted italic); blank-line-separated paragraphs (15/26
  textMuted).
- **Inline** (within non-code text): `**bold**` (bodyMedium, text color),
  `*italic*` (fontStyle italic), `` `code` `` (mono 13.5, surfaceStrong bg,
  radius 4, px 5), `[text](url)` (accent, underline on web hover →
  `Linking.openURL`). Parser: single regex tokenizer over the segment,
  rendered as nested `<Text>` runs.

- `src/admin/components/posts-view.tsx`:
  - **List mode**: ViewHeader `Blog` / `N posts`; `+ Nuevo post` AccentButton;
    rows (surface card): status badge (PUBLICADO accent-tinted / BORRADOR

    `Editar` chip + `Eliminar` HoverLink (confirm via two-tap: first tap turns
    the link into `¿Seguro?`).
  - **Editor mode**: `← Volver` HoverLink; Fields: `slug` (auto-slugified from

    Borrador/Publicado); `— Español —` Label + title/excerpt Fields + content
    Field multiline; `— English —` same. SaveBar-style row: status text +
    `💾 Guardar` AccentButton → `savePost`, back to list on success. Reminder
    text under save: `Recuerda Publicar para verlo en vivo.`

### Seed

`scripts/migrate-blog-seed.ts`: creates `posts/bienvenido-al-blog` (published,
publishedAt today, tags `['portfolio', 'detrás de cámaras']`), ES/EN welcome
post (~6 short markdown paragraphs with a heading, a list and a code fence so
the renderer is exercised); skips if the doc already exists (no overwrite).
Then the published mirror `posts.json` is written locally (running
`content:pull` needs full content — fine locally with ADC/service account).

## Error handling

- Unknown `/blog/<slug>` at runtime (stale link): redirect effect to `/blog`.
- posts.json empty: list shows the empty state; `generateStaticParams`
  returning `[]` emits no detail pages (verified in build).
- Markdown renderer treats unknown syntax as plain paragraph text (never
  throws).
- Admin save with empty slug/es.title: SaveBar shows `Error: slug y título ES
  son obligatorios` (client-side guard, no write).
